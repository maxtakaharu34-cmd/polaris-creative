# 画像フォルダ運用ルール

このフォルダに画像を置けば、コードから即座に読み込めます。

## フォルダ構成

| フォルダ | 用途 | 推奨サイズ | 使用箇所 |
|---|---|---|---|
| `hero/` | ヒーロースライダー（トップ最上部） | 2000×1200px (横長) | 自動切替スライド |
| `works/` | 施工実績ギャラリー | 1200×800px | 実績一覧・モーダル |
| `team/` | 代表・スタッフ・職人 | 800×800〜1200×800px | 会社案内・代表挨拶 |
| `recruit/` | 採用ページ用（現場・チーム） | 1600×900px | 採用CTAバナー |
| `logo/` | ロゴ・ファビコン | SVG推奨 / 512×512px (PNG) | ヘッダー・タブアイコン |
| `og/` | SNSシェア用OG画像 | 1200×630px | TwitterCard・Facebook |

## ファイル命名規則

- 連番: `hero-01.jpg`, `hero-02.jpg`, `work-01.jpg` ...
- 拡張子は `.jpg`（写真）/ `.png`（透過必要時）/ `.webp`（軽量化したい時）
- 半角英数 + ハイフン推奨（日本語ファイル名は避ける）

## 推奨フォーマット

- **写真** → JPG (品質80%) または WebP
- **ロゴ/アイコン** → SVG > PNG透過
- **大きすぎる場合** → 各推奨サイズ以下にリサイズしてから配置

## 使い方

ファイルをこのフォルダに置いた後、Claude に伝えるだけ：
> 「`hero-01.jpg` を1枚目スライドに使って」
> 「`works/work-03.jpg` を施工実績の3つ目に」

コード側は `/images/hero/hero-01.jpg` のような絶対パスで参照します。
